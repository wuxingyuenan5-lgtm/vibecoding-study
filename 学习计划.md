# Vibe Coding 学习计划

## 总判断

原学习指南方向正确，但对代码小白节奏偏快。最优路线不是三个仓库并行学，而是一个主项目反复迭代。

核心安排：

```text
easy-vibe-main 做主线
vibe-coding-cn-develop 做工作流参考
awesome-vibe-coding-main 做工具索引
练习代码只放进 练习文件夹
```

学习目标不是成为传统程序员，而是先成为能把想法稳定做成可运行原型的人。

## 你的真实学习目标

你当前不是纯兴趣学习，而是在做一个接近完成的私募基金平台业务。你已经有较明确的业务需求，真正短板在：

1. 不熟悉工程化表达。
2. 不熟悉程序员常用术语。
3. 不熟悉项目拆解、版本控制、质量验收。
4. 不熟悉如何约束 AI 修改范围。
5. 不熟悉如何把业务需求翻译成页面、字段、接口、状态、权限和验收标准。

因此，后续学习要从“玩具项目”尽快过渡到“金融业务相关的低风险模块”。小游戏只用于训练 AI 编程闭环，不作为长期方向。

## 第 0 周：准备期

周期：1-2 天

目标：确认本地开发动作能走通。

任务：

1. 确认 Node.js、Git、浏览器、编辑器/Codex 可用。
2. 学会进入目录、安装依赖、启动项目、停止项目。
3. 在 `练习文件夹` 创建第一个练习项目目录。
4. 建立 `学习日志.md`。

必须掌握的命令概念：

```text
cd
npm install
npm run dev
Ctrl + C
git status
```

验收标准：

- 能打开一个本地网页。
- 知道终端报错在哪里看。
- 知道项目文件夹和资料库文件夹不能混在一起。

## 第 1 周：第一个小游戏

主线资料：`easy-vibe-main`

建议项目：`snake-game-01`

目标：完成第一个“能玩、能改、能修”的小游戏。

功能范围：

1. 方向键控制。
2. 计分。
3. 撞墙或撞自己失败。
4. 开始和重新开始。
5. 简洁 UI。

学习重点：

- 不要求代码全懂。
- 要知道项目怎么启动。
- 要知道改动后怎么验证。
- 要经历一次真实 Debug。

验收标准：

- 游戏能在浏览器运行。
- 至少成功加过 2 个小功能。
- 至少修过 1 个 bug。
- 学习日志里记录一次完整问题。

## 第 2 周：第一个实用工具

主线资料：`easy-vibe-main`

辅线资料：`vibe-coding-cn-develop` 的 Prompt / workflow 内容

建议项目：`trade-journal-01`

目标：做一个交易复盘日志工具。它贴近私募基金平台业务，但不碰真实账户、真实客户和生产数据。

第一版功能：

1. 新增交易记录。
2. 展示交易记录列表。
3. 删除交易记录。
4. 刷新页面后数据不丢。

字段建议：

```text
日期
标的
方向
入场理由
退出理由
盈亏
情绪评分
复盘备注
```

学习重点：

- 把一句话需求改写成结构化需求。
- 让 AI 先出计划，再写代码。
- 开始理解数据结构。

验收标准：

- 能新增、查看、删除记录。
- 使用 localStorage 保存。
- 能说清楚每个字段为什么存在。

## 第 3-4 周：把工具做得像产品

主项目：继续迭代 `trade-journal-01`

不要新开项目。

升级功能：

1. 编辑已有记录。
2. 按标的筛选。
3. 按方向筛选。
4. 统计总交易数、胜率、总盈亏、平均盈亏。
5. 增加盈亏曲线或柱状图。
6. 加空状态和错误提示。
7. 适配手机屏幕。

学习重点：

- 从 Demo 进入产品原型。
- 学会验收 UI、交互、数据是否正确。
- 学会控制 AI 修改范围。

每次改动前固定问：

```text
你准备改哪些文件？
这个改动可能破坏什么？
改完后怎么验证？
```

验收标准：

- 项目连续迭代 5 次以上仍能运行。
- 没有因为乱改导致整体不可用。
- 有清晰学习日志。

## 第 5-6 周：建立个人工作流

主线资料：`vibe-coding-cn-develop`

只学四个概念：

1. Prompt
2. Context
3. Quality Gate
4. Git

本阶段产出 5 个自己的模板：

1. 新功能模板
2. 修 bug 模板
3. UI 优化模板
4. 代码解释模板
5. 项目复盘模板

学习重点：

- 让 AI 输出更稳定。
- 每次任务都有边界。
- 每次完成都有验收。
- 开始用 Git 保存稳定版本。

验收标准：

- 能用模板发起任务。
- 能让 AI 解释代码。
- 能用 `git status` 查看变更。
- 能保存至少 3 个稳定版本。

## 第 7-8 周：工具选择与部署

参考资料：`awesome-vibe-coding-main`

目标：只选工具，不收藏工具。

只关注三类：

1. AI 编程工具：Codex / Cursor / Claude Code 选一个主力。
2. 部署工具：Vercel 或 Netlify 选一个。
3. 数据工具：Supabase 先了解，不急着重度使用。

项目目标：

把 `trade-journal-01` 或升级版部署成可访问网页。

注意：

- 不上传真实交易数据。
- 不提交 API Key。
- 部署前用模拟数据。

验收标准：

- 有一个可打开的网址。
- 能说明本地运行和线上部署的区别。
- 知道哪些数据不能放到公开项目。

## 两个月后的目标作品

推荐作品：

```text
私募基金平台内部练习模块
```

建议从 `交易复盘日志 Web App` 开始。它应该具备：

1. 新增、编辑、删除交易记录。
2. 本地保存数据。
3. 筛选和统计。
4. 简单图表。
5. 移动端可用。
6. 有学习日志。
7. 有 README。

这个作品比做三个教程 Demo 更有价值，因为它贴近你的真实使用场景，也能训练持续维护能力。

后续可以逐步扩展到：

1. 研究笔记管理。
2. 标的池跟踪。
3. 策略观察列表。
4. 风控指标看板原型。
5. 投资决策会议纪要模板。

这些都先用模拟数据做，训练工程能力，不连接生产系统。

## 何时再学 Skill / Agent

满足以下条件后再学：

- 已经完成一个可运行项目。
- 已经连续迭代 5 次以上。
- 已经沉淀出自己的 Prompt 模板。
- 已经知道哪些任务会反复出现。

否则不要急着做 Skill。过早做 Skill，只是在包装还不成熟的方法。

## 每周复盘问题

每周末回答：

1. 本周最关键的一个突破是什么？
2. 哪个错误重复出现？
3. 哪个 Prompt 最有效？
4. 哪个需求其实没说清楚？
5. 下周只做哪一个主项目？
