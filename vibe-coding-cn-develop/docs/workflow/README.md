# workflow

## 字多不看

- 本目录收敛项目开发流程，回答“从接到任务到提交推送应该怎么做”。
- 默认流程是明确目标、读取上下文、制定计划、执行修改、运行门禁、检查差异、控制版本、推送远端、同步文档。
- 流程要能执行、检查和复用，不写只适合一次性任务的日志。

## 使用方式

- 开始任务前，先按开发流程确认任务顺序和验收节点。
- 需要执行 Git、提交或推送时，同时遵循根目录 AGENTS.md 中的版本控制规则。
