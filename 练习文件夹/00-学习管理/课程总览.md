# Vibe Coding 课程总览

## 一、课程定位

本课程面向：

- 代码基础较弱，但已经有明确业务需求的学习者。
- 具备金融研究、交易与产品思维，希望借助 AI 独立推进复杂项目。
- 目标不是转型为传统全职程序员，而是成为能够设计、拆解、监督、验收和持续维护 AI 软件项目的人。

核心目标：

```text
业务想法
→ 工程规格
→ AI 执行任务
→ GitHub 变更
→ 运行与验证
→ Debug
→ 迭代与复盘
```

课程不会长期停留在玩具项目，也不会直接进入真实生产系统。所有练习优先采用与研究、交易、基金平台、数据看板相关的低风险模拟场景。

## 二、资料库分工

| 资料库 | 使用方式 |
|---|---|
| `easy-vibe-main` | 新手概念、基础项目和直观示例 |
| `vibe-coding-cn-develop` | Prompt、Context、Quality Gate、Git、工程流程 |
| `awesome-vibe-coding-main` | 需要选择工具时查询，不作为主线教材 |

原则：按当前课程需要定向读取，不并行通读三个资料库。

## 三、主线课程规划

| 课次 | 课程主题 | 核心能力 | 主要产出 |
|---|---|---|---|
| 第 0 课 | 环境与 GitHub 闭环 | 仓库、提交、推送、分支、PR 的基本概念 | 学习仓库与双聊天协作方式 |
| 第 1 课 | 从业务想法到工程规格 | 用户、问题、范围、状态、字段、流程、验收标准 | 研究笔记看板 MVP 规格书 |
| 第 2 课 | Git、分支、PR、Diff 与回滚 | 看懂改动、隔离风险、保存稳定版本 | 完成一次规范 PR 审查闭环 |
| 第 3 课 | 看懂 Web 项目的骨架 | HTML、CSS、JavaScript、入口、依赖、运行命令 | 静态金融信息卡片页面 |
| 第 4 课 | 状态、事件与渲染 | 前端交互的底层模型 | 可交互标的观察列表 |
| 第 5 课 | 数据模型、CRUD 与本地保存 | 字段设计、增删改查、localStorage | 交易或研究复盘工具 MVP |
| 第 6 课 | Debug 与浏览器开发者工具 | 复现、定位、日志、最小修复、回归检查 | 一次完整真实 Bug 复盘 |
| 第 7 课 | API、异步、错误处理与密钥边界 | fetch、loading/error/empty、环境变量 | 使用模拟或公开数据的看板 |
| 第 8 课 | 页面架构、组件与 UI 系统 | 页面拆分、复用、设计一致性、响应式 | 产品化内部工具界面 |
| 第 9 课 | 数据库、登录与权限模型 | 表结构、鉴权、角色权限、隐私边界 | 模拟内部平台模块 |
| 第 10 课 | 测试与 Quality Gate | 单元测试、集成测试、类型、Lint、验收证据 | 自动化质量检查清单 |
| 第 11 课 | 部署、CI/CD、安全与可观测性 | 本地/测试/生产环境、日志、回滚 | 可访问的演示版本 |
| 第 12 课 | Agentic Engineering 与复杂项目管理 | AGENTS、计划、项目记忆、子任务、隔离审查 | 个人 AI 工程工作流模板 |
| 综合项目 | 私募基金平台低风险模块 | 综合运用需求、架构、数据、测试、部署 | 研究笔记、策略观察或风控看板模块 |

## 四、项目安排

### 可选热身

`01-snake-game-01` 保留为可选热身项目，用于快速理解状态、事件和游戏循环，不再作为正式主线。

### 正式主线

正式课程统一创建在：

```text
练习文件夹/主线课程/
```

每节课使用独立目录，但中后期会围绕同一个业务型项目持续迭代，避免产生大量互不相关的半成品。

## 五、每节课的固定文件

每节正式课程至少包含：

```text
README.md
01-教学讲义.md
02-课堂作业.md
03-执行任务卡.md
04-验收清单.md
05-学习日志.md
06-老师评语.md
```

代码文件根据该课实际任务创建，不强制使用固定技术栈。

## 六、教学与执行分工

- **老师聊天**：设计课程、讲解知识、维护讲义、布置作业、批准任务、审查 PR、决定是否进入下一课。
- **AI 执行聊天**：只按批准的任务卡修改授权文件、创建分支和 PR，不负责扩展需求或最终验收。
- **学习者**：理解、判断、回答作业问题；需要真实界面或运行结果时，在本地执行并反馈。

## 七、默认教学原则

1. 每节课先理解，再执行；不能把全部思考外包给 AI。
2. 一次只解决一个明确问题。
3. 先写范围、非目标和验收标准，再生成代码。
4. AI 输出是候选解，运行结果、Diff 和验收证据才是裁决依据。
5. 不因“更先进”或“更优雅”随意换框架、目录和技术栈。
6. 所有真实账户、客户数据、交易数据、密钥和生产接口均不得进入公开练习仓库。
7. 课程解释优先贴近真实业务，但不会把所有普通编程概念强行金融化。
8. 每节课的新增知识、用户补充、纠错和重要讨论都必须同步回对应的 `01-教学讲义.md`。
