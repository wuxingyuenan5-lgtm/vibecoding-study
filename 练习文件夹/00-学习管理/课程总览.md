# Vibe Coding 课程总览

## 教学目标

把业务想法稳定转化为可运行、可修改、可验证、可回滚的项目，而不是只会让 AI 一次性生成代码。

## 学习路线

| 阶段 | 课程 | 核心能力 | 主要产出 |
|---|---|---|---|
| 第 0 阶段 | 环境与 Git 闭环 | 目录、终端、运行、提交、推送 | 本仓库初始化完成 |
| 第 1 阶段 | 贪吃蛇小游戏 | 文件职责、状态、事件、Debug | `01-snake-game-01` |
| 第 2 阶段 | 交易复盘工具 | 需求结构化、数据模型、localStorage | `02-trade-journal-01` |
| 第 3 阶段 | 产品化迭代 | 筛选、统计、编辑、图表、移动端 | 持续迭代同一项目 |
| 第 4 阶段 | 个人工作流 | Prompt、Context、Quality Gate、Git | 个人模板库 |
| 第 5 阶段 | 部署 | 本地与线上环境、隐私边界 | 可访问的演示网页 |

## 当前课程

- 当前阶段：第 1 阶段
- 当前项目：`01-snake-game-01`
- 当前任务：先完成实施计划，经审查后再编写第一版代码

## 教学原则

1. 一次只推进一个主项目。
2. 先跑通，再优化。
3. 小步提交，每次提交只表达一个明确变化。
4. AI 输出是候选解，必须通过运行结果和验收清单裁决。
5. 不为“更优雅”无理由换框架、换目录或大规模重构。
