# 通用术语表

## 项目与文件

| 术语 | 含义 |
|---|---|
| Repository / 仓库 | 保存项目文件、版本历史和协作记录的目录 |
| Project / 项目 | 为解决一个明确问题而组织的一组代码和文档 |
| Root / 根目录 | 项目的最外层目录 |
| Entry / 入口 | 程序开始运行或页面首先加载的位置 |
| Dependency / 依赖 | 项目运行所需要的外部库或工具 |

## 开发流程

| 术语 | 含义 |
|---|---|
| Requirement / 需求 | 用户希望系统实现的行为和结果 |
| Scope / 范围 | 本次任务包含与不包含的内容 |
| Implementation Plan / 实施计划 | 编码前说明修改文件、方案、风险与验证方式 |
| Acceptance Criteria / 验收标准 | 判断任务是否完成的可检查条件 |
| Debug | 定位并修复程序错误的过程 |
| Refactor / 重构 | 不改变外部功能，调整内部代码结构 |
| Regression / 回归问题 | 新改动破坏原本正常功能 |

## Web 基础

| 术语 | 含义 |
|---|---|
| HTML | 定义页面内容和结构 |
| CSS | 控制页面布局和视觉样式 |
| JavaScript | 控制交互、状态和业务逻辑 |
| DOM | 浏览器把 HTML 表示成可被 JavaScript 操作的对象结构 |
| Event / 事件 | 用户按键、点击等触发的信号 |
| State / 状态 | 程序某一时刻保存的数据，例如分数、蛇的位置、游戏是否结束 |
| Render / 渲染 | 根据当前状态把结果显示到页面 |

## Git 基础

| 术语 | 含义 |
|---|---|
| Working Tree | 本地当前文件状态 |
| Stage / 暂存区 | 准备放入下一次提交的改动 |
| Commit / 提交 | 一次有说明、可追踪的版本快照 |
| Branch / 分支 | 从某个版本独立发展的修改线 |
| Push | 把本地提交上传到 GitHub |
| Pull | 把远程更新同步到本地 |
| Diff | 两个版本之间的具体差异 |
