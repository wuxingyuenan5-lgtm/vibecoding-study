# 练习区说明

本目录用于保存正式课程、作业、实践代码、验收记录和复盘文档。

## 目录结构

```text
练习文件夹/
├─ 00-学习管理/
│  ├─ 课程总览.md
│  ├─ 学习进度.md
│  ├─ 通用术语表.md
│  ├─ 教师端默认规则.md
│  └─ AI执行端指令.md
├─ 主线课程/
│  └─ 01-业务想法到工程规格/
│     ├─ README.md
│     ├─ 01-教学讲义.md
│     ├─ 02-课堂作业.md
│     ├─ 03-执行任务卡.md
│     ├─ 04-验收清单.md
│     ├─ 05-学习日志.md
│     └─ 06-老师评语.md
└─ 01-snake-game-01/
   └─ 可选热身项目
```

## 固定学习闭环

1. 老师建立课程目录、教学讲义、课堂作业和验收标准。
2. 学习者在老师聊天中理解概念并完成需要本人判断的作业。
3. 老师审查答案，把有效补充、纠错和讨论同步到对应课程的 `01-教学讲义.md`。
4. 老师批准实施方案并形成 `03-执行任务卡.md`。
5. AI 执行端只在授权分支中修改指定文件，并创建 Pull Request。
6. 老师审查 Diff、代码和文档，必要时要求修正。
7. 需要真实运行或界面判断时，由学习者在本地运行并反馈结果。
8. 老师完成验收、评语和学习进度更新。

## 文档归档原则

- 每节正式课程必须有独立课程文件夹和 `01-教学讲义.md`。
- 聊天中的重要新增知识不能只停留在聊天记录，必须同步进对应讲义。
- 课堂作业、执行任务、验收结论和老师评语分别归档，不相互混写。
- 同一知识点的补充内容优先更新原课程讲义，不额外创建零散笔记。

## 文件边界

- 三个资料库只用于阅读和查找资料，不直接写入练习代码。
- 每个课程项目都使用独立文件夹。
- 真实账户、客户隐私、API Key、生产数据不得提交。
- 所有金融业务练习默认使用模拟数据和脱敏字段。
- AI 执行端不得直接修改 `main`，不得未经授权合并 PR。
