# 第 1 课学习日志

## 2026-07-23

### 本课目标

学会把长期愿景拆成产品项目、阶段目标、课程练习和单次执行任务，并进一步形成可执行、可审查、可验收的工程规格。

### 我的五层任务地图

```text
长期愿景：
形成平台化、自动化、AI 驱动的个人投研工作体系。

产品项目：
将幕布笔记逐步迁移到 Obsidian，并建设 AI 投研知识库。

当前阶段目标：
完成少量幕布笔记的可验证迁移，确认正文、层级、标签和附件能够保留。

本课练习：
使用三份模拟投研笔记，理解 Markdown 正文与 YAML 元数据的分工。

单次执行任务：
在学习仓库的指定目录中新增三份模拟 Markdown 笔记；
每份包含 title、createdAt、tags 三个 YAML 字段；
不编写迁移工具，不修改真实产品仓库。
```

### 当前理解

- 长期愿景可以很大，但不能直接交给 AI 执行。
- 产品项目承载长期愿景。
- 阶段目标可以覆盖多个验证维度，不要求一个任务全部完成。
- 课程练习只训练一个主要能力。
- 单次执行任务必须限制目录、文件和非目标。

### 老师纠正了什么

1. 当前阶段目标包含正文、层级、标签和附件四个验证维度，属于一个阶段，不应被理解为一次任务。
2. 本课练习只训练 Markdown 正文与 YAML 元数据的分工，因此不需要同时处理层级和附件。
3. 单次任务只写 YAML 字段还不够；三份样例还应包含实际 Markdown 正文，否则无法观察正文与元数据的分工。
4. 本轮不覆盖附件和层级并不是遗漏，而是主动控制 Scope。
