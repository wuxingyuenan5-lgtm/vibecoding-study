# 第 1 课：从业务想法到工程规格

## 课程定位

本课训练 Vibe Coding 最重要的前置能力：把“我想做一个功能”翻译成 AI 可以执行、老师可以审查、学习者可以验收的工程规格。

## 练习案例

本课使用模拟项目：

```text
研究笔记看板 MVP
```

该案例贴近研究工作，但只使用模拟数据，不连接真实账户、客户、交易系统或生产数据库。

## 通过标准

学习者能够独立说清：

- 谁在使用产品、解决什么问题。
- 第一版做什么和不做什么。
- 用户如何操作。
- 需要保存哪些字段和状态。
- 如何用可执行的验收标准判断是否完成。
- 哪些内容可以在线审查，哪些必须本地运行。
