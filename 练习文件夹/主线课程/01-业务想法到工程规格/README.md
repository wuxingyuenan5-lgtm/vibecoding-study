# 第 1 课：从业务想法到工程规格

## 课程定位

本课训练 Vibe Coding 最重要的前置能力：把“我想做一个功能”翻译成 AI 可以执行、老师可以审查、学习者可以验收的工程规格。

## 练习案例

本课案例已经根据学习者的真实需求调整为：

```text
开放格式、AI 友好的个人投研知识库 MVP
```

核心用户是同时承担研究和交易的个人用户。案例重点解决研究笔记分散、封闭软件难以接入 AI、资料不便迁移和检索的问题。

第一版只建立知识库基础，不直接实现每日信息自动采集、AI 自动分类、实时行情、云数据库或完整投研平台。

## 当前状态

## 通过标准

学习者能够独立说清：

- 谁在使用产品、解决什么问题。
- 自动信息整理、知识库存储和 AI 应用属于不同系统层。
- 第一版做什么和不做什么。
- 用户如何操作。
- 需要保存哪些字段、附件信息和状态。
- “开放格式”和“可迁移”具体意味着什么。
- 如何用可执行的验收标准判断是否完成。
- 哪些内容可以在线审查，哪些必须本地运行。
