# 第 1 课：从大愿景到可执行小任务

## 课程定位

本课训练 Vibe Coding 最重要的前置能力：把庞大的长期愿景拆成 AI 可以执行、老师可以审查、学习者可以验收的小任务。

## 学习者真实背景

当前存在三个不同方向：

1. 幕布笔记逐步迁移到 Obsidian，并发展为 AI 投研知识库。
2. `wuxingyuenan5-lgtm/news-stugy` 持续开发信息自动搜集和整理能力。
3. `wuxingyuenan5-lgtm/Platform_Experiment` 已经是复杂基金与交易研究平台，用于强化高级 Vibe Coding 能力。

这三个方向都是真实项目，但第一课不要求立即完成其中任何一个产品。

## 本课核心问题

必须区分五个层级：

```text
长期愿景
→ 产品项目
→ 阶段目标
→ 课程练习
→ 单次执行任务
```

例如：

```text
长期愿景：全面平台化、AI 投研化
产品项目：Obsidian / AI 投研知识库
阶段目标：建立稳定的 Markdown 和 YAML 规范
课程练习：为三条模拟笔记设计统一字段
执行任务：只新增一个样例目录和三份 Markdown 样例
```

## 通过标准

学习者能够独立：

- 区分长期愿景、产品项目、阶段目标、课程练习和执行任务。
- 解释为什么真实产品不能直接作为单次 Prompt。
- 从大型目标中切出一个小而完整的任务。
- 写清用户、问题、范围、非目标和验收标准。
- 判断某项练习是否应该进入真实产品仓库。
- 区分能力学习、产品产出和高级案例审查。
