# 第 1 课课堂作业

## 作业目标

不写代码。把长期愿景、三个真实项目和一次微型执行任务分开，形成第一份边界清晰的工程规格。

老师会在聊天框中逐项审查，并把长期有效的结论同步进本课讲义或学习者项目地图。

## 回答要求

- 使用自己的判断，不让 AI 代替全部思考。
- 不需要掌握技术术语，但必须把实际意思说清楚。
- 不以“功能越多越好”为目标。
- 当前不修改 `news-stugy` 或 `Platform_Experiment`。
- 当前不迁移真实幕布数据。

## 题目 1：五层任务地图

请分别填写：

```text
长期愿景：
产品项目：
当前阶段目标：
本课练习：
可能的单次执行任务：
```

可以选择 Obsidian / AI 投研知识库或 `news-stugy` 作为案例，但只选一个。

## 题目 2：为什么不能直接开发完整愿景

请说明：

1. 直接要求 AI 完成完整产品时，最可能出现的三个问题。
2. 为什么“先做一个小练习”并不等于偏离长期目标。
3. 什么情况下课程练习应该进入真实产品仓库，什么情况下应该留在学习仓库。

## 题目 3：选择一个微型练习

从以下选一个：

### A. Markdown 元数据校验练习

```text
读取三份模拟 Markdown
→ 检查 YAML 中的 title、createdAt 和 tags
→ 输出校验结果
```

### B. 新闻输入校验练习

```text
读取一份模拟 JSON 新闻数组
→ 检查 title、source 和 published_at
→ 输出缺失字段
```

### C. 你自己提出一个同等规模练习

要求只包含一个输入、一个主要处理步骤和一个明确输出。

请说明为什么选择它，以及它主要训练什么能力。

## 题目 5：验收标准

