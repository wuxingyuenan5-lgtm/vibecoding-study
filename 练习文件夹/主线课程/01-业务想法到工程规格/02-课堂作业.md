# 第 1 课课堂作业

## 作业目标

不写代码，独立完成“研究笔记看板 MVP”的第一版业务规格。

老师会根据你的回答进行纠错，并把有长期价值的补充同步进本课 `01-教学讲义.md`。

## 回答要求

- 使用自己的判断，不要让 AI 代答全部内容。
- 可以不懂技术名词，但要把业务意思说清楚。
- 每题尽量具体，避免“简单、方便、智能、专业”等无法验收的词。
- 当前不考虑真实数据、登录、权限、行情和生产系统。

## 题目 1：主要用户与问题

请回答：

1. 第一版主要给谁使用？只能选一个核心用户角色。
2. 这个用户目前管理研究笔记时最具体的 2—3 个问题是什么？
3. 哪一个问题是第一版必须优先解决的？

## 题目 2：第一版目标

请用一句话完成：

```text
第一版要让【谁】能够在【什么场景】完成【什么核心动作】，从而解决【什么问题】。
```

## 题目 3：核心操作

请从下列操作中选出第一版必须具备的 3—5 项，并说明理由：

- 新增笔记
- 查看列表
- 查看详情
- 编辑
- 删除
- 按标的筛选
- 按状态筛选
- 按日期筛选
- 搜索关键词
- 排序
- 导出
- 统计

也可以提出未列出的操作，但需要说明为什么是第一版必需。

## 题目 5：状态设计

当前建议状态是：

```text
跟踪中 / 已验证 / 已失效
```

请回答：

1. 这三个状态是否够用？
2. 每个状态分别代表什么业务含义？
3. 状态由用户手动修改，还是由系统自动判断？为什么？

## 题目 6：用户流程

请用箭头写出一个完整主流程，例如：

```text
打开页面 → …… → …… → 完成目标
```

至少覆盖：新增、找到一条已有笔记、修改状态。

## 题目 7：第一版非目标

请明确列出至少 5 项本轮不做的内容，并说明其中最容易诱发范围膨胀的是哪一项。

## 题目 8：验收标准

请写出至少 6 条可操作验证的验收标准。

每条尽量使用：

```text
前提 + 操作 + 预期结果
```

至少覆盖：

- 新增成功
- 必填项校验
- 筛选
- 编辑
- 删除确认
- 刷新后数据保留
- 空状态或无匹配结果

## 题目 9：在线与本地验证

请分别列出：

1. 仅通过 GitHub 文件和 Diff 就能检查的内容。
2. 必须在浏览器中实际运行才能确认的内容。
