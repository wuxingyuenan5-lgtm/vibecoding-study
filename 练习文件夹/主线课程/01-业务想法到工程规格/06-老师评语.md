## 最新背景确认

学习者的长期方向是平台化、自动化和 AI 投研化，当前有三个不同层级的真实方向：

1. 幕布笔记全面迁移到 Obsidian，并逐步建设 AI 投研知识库。
2. `wuxingyuenan5-lgtm/news-stugy` 正在开发信息自动搜集、整理和 Markdown 日报能力。
3. `wuxingyuenan5-lgtm/Platform_Experiment` 已经是复杂基金与交易研究平台。

课程目标是强化扎实、可迁移的 Vibe Coding 基本功，而不是让每节课都直接推动上述大型产品。

2. Obsidian / AI 知识库和信息自动搜集整理可以成为学习过程中的产品产出。
3. 基金平台不需要在课程中重复建设，重点是学习如何理解、审查和安全维护复杂项目。
4. 独立小练习是必要训练，不要求每个练习都写入真实产品。
5. 第一课已经从“完整知识库 MVP 规格”调整为“长期愿景到微型任务的层级拆解”。

## 第二轮：五层任务地图审查

### 学习者答案

```text
当前阶段目标：
完成少量幕布笔记的可验证迁移，确认正文、层级、标签和附件能够保留。

本课练习：
使用三份模拟投研笔记，理解 Markdown 正文与 YAML 元数据的分工。

单次执行任务：
在学习仓库的指定目录中新增三份模拟 Markdown 笔记，
每份包含 title、createdAt、tags 三个 YAML 字段；
不编写迁移工具，不修改真实产品仓库。
```

### 结论

`层级判断通过，单次任务需小幅补全`

### 做得好的地方

1. 阶段目标、课程练习和单次任务的尺度已经明显区分。
2. 将真实幕布迁移放在阶段目标，而不是直接塞入本轮任务，范围控制正确。
3. 本轮明确使用模拟投研笔记，避免真实数据和迁移风险。
4. 明确不编写迁移工具、不修改真实产品仓库，Non-goal 意识已经建立。
5. 单次任务文件数量少、结果可直接通过 GitHub Diff 审查，适合作为第一次执行任务。

### 必须修正的地方

1. 阶段目标中的“正文、层级、标签、附件”是四个不同验证维度，不能由这一次任务全部证明。
2. 本课练习的目标是理解 Markdown 正文与 YAML 元数据的分工，因此三份样例必须同时有 YAML 和实际 Markdown 正文。
3. `title`、`createdAt`、`tags` 只定义了元数据，没有定义正文最少包含什么。
4. 本轮不处理层级和附件不是功能缺失，而是主动缩小 Scope。后续可以分别建立独立任务验证。
5. 当前任务还缺少具体目录、样例主题、允许新建文件、验收标准和人工检查方式，因此暂不交给执行端。

### 建议采用的任务表述

```text
在学习仓库的指定练习目录中新增一个 README 和三份模拟投研 Markdown 笔记。

每份笔记：
- YAML Front Matter 包含 title、createdAt、tags；
- Markdown 正文至少包含两个二级标题和对应的模拟内容；
- 三份笔记使用不同主题和标签，便于人工比较。

本轮不编写迁移工具，不处理真实幕布数据，不处理附件和目录层级，不接入 Obsidian 插件或 AI，不修改 news-stugy 和 Platform_Experiment。
```
