# 第 1 课作业要求

## 作业目标

完成一个可在浏览器运行的网页版贪吃蛇，并通过文档记录计划、验收和复盘过程。

## 阶段 A：实施计划

当前只完成阶段 A，暂不编写游戏代码。

请在 `03-实施计划.md` 中写清：

1. 准备创建哪些代码文件。
2. 每个文件的职责。
3. 游戏需要保存哪些状态。
4. 游戏启动、移动、吃食物、失败和重开的流程。
5. 最可能出现的 3 个问题。
6. 每项功能如何验证。
7. 第一版明确不做哪些功能。

完成后提交并推送 GitHub，等待老师审查。

## 阶段 B：第一版实现

计划通过后再实现以下功能：

1. 方向键控制蛇移动。
2. 吃到食物后蛇增长，分数增加。
3. 撞墙或撞到自己后游戏结束。
4. 有开始和重新开始按钮。
5. 页面显示当前分数和游戏状态。
6. 使用原生 HTML、CSS 和 JavaScript。
7. 双击 `index.html` 或通过简单本地服务器即可运行。

## 第一版非目标

以下内容本课不做：

- React、Vue 或其他前端框架
- 后端和数据库
- 登录、排行榜和联网功能
- 手机触屏控制
- 音效、复杂动画和皮肤系统
- 难度选择和关卡系统
- 大规模目录设计或通用游戏引擎

## 提交要求

每个阶段单独提交，建议提交说明：

```text
docs: complete snake game implementation plan
feat: implement snake game MVP
fix: correct snake collision logic
docs: complete lesson 1 review
```

禁止把真实账号、隐私数据、密钥或无关大文件提交到仓库。
