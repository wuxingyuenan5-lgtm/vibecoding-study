# Vibe Coding 学习项目规则

## 项目定位

这是代码新手的 Vibe Coding 练习区。目标不是快速堆功能，而是稳定建立：

```text
想法 -> 需求 -> AI 生成 -> 本地运行 -> Debug -> 小步迭代 -> 复盘沉淀
```

学习者背景：

- 代码小白。
- 正在初创私募基金平台业务。
- 对业务需求较明确，但缺少工程化、程序员术语、项目管理和质量验收经验。
- 学习目标是提升专业 Vibe Coding 能力，而不是转型为传统全职程序员。

后续教学和练习要尽量贴近私募基金平台相关场景，但早期必须避开真实资金、真实账户、真实客户数据、真实 API Key 和生产系统。

## 三个资料库的分工

`easy-vibe-main` 是主线教材和项目来源，优先用于新手阶段。

`vibe-coding-cn-develop` 是工作流、Prompt、Context、Quality Gate、Skill 的资料库。早期只查 Prompt 和工作流，不逐章硬啃。

`awesome-vibe-coding-main` 是工具索引。只在需要选工具时查，不作为学习主线。

## 学习原则

1. 一次只做一个主项目。
2. 先跑通，再优化。
3. 先改小功能，再做大重构。
4. 每次改动前必须说清楚验收标准。
5. 不追求“AI 一次生成完美项目”，重点训练发现问题和修问题。
6. 不复制粘贴一堆看不懂的代码后继续加需求。
7. 遇到报错时优先复制完整终端输出、运行命令、最近改动。

## AI 执行规则

每次让 AI 修改代码前，要求 AI 先回答：

```text
你准备改哪些文件？
为什么要改这些文件？
这个改动可能破坏什么？
改完后我应该如何验证？
```

如果 AI 准备大范围重构，必须先暂停，让它解释必要性。新手阶段默认禁止无理由重构。

如果项目已经能跑，不允许为了“更优雅”随意换框架、换构建工具、换目录结构。


## 阶段验收标准

第 1 阶段合格：

- 能启动本地项目。
- 能看懂哪个页面是入口。
- 能完成一个小游戏。
- 能让 AI 修复至少一个真实 bug。

第 2 阶段合格：

- 能完成一个本地 Web 工具。
- 能用 localStorage 保存数据。
- 能自己写结构化需求。
- 每次改动都有验收标准。
- 能把业务语言翻译成工程任务，例如页面、字段、状态、权限、数据来源、验收标准。

第 3 阶段合格：

- 能维护一个持续迭代的项目。
- 能做筛选、统计、编辑、删除。
- 能用 Git 保存稳定版本。
- 能判断 AI 是否在无意义重构。
- 能区分 Demo、原型、内部工具、生产系统的质量要求。

## 禁止事项

禁止批量删除文件或目录。

不要使用：

```text
del /s
rd /s
rmdir /s
Remove-Item -Recurse
rm -rf
```

需要删除文件时，只能一次删除一个明确路径的文件。

不要把真实隐私数据、真实 API Key、真实交易账户信息提交到项目里。

涉及私募基金平台、交易、客户、账户、净值、风控、合规等内容时，默认只使用模拟数据和脱敏字段。
