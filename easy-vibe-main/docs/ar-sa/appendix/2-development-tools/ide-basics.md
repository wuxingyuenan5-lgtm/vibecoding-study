# أساسيات بيئة التطوير المتكاملة (IDE)

::: tip 💡 دليل التعلم
سيأخذك هذا الفصل في جولة معمّقة لأهم أداة إنتاجية للمبرمجين — **بيئة التطوير المتكاملة (IDE)**. سننطلق من فلسفة تصميم الـ IDE، ونحلل مكوناته الأساسية واحداً تلو الآخر، ومن خلال IDE افتراضي سنوضح كيف يعمل.
:::

## ماذا تفعل عندما لا تفهم شيئاً؟ (How to solve problems)

أثناء تعلم واستخدام الـ IDE، قد تواجه أزراراً أو قوائم أو أخطاء في الكود لا تفهمها. في هذه الحالة، **لا تهلع؛ استخدام مساعد الذكاء الاصطناعي هو الحل الأكثر كفاءة**.

**التوصية: التقط لقطة شاشة واسأل الذكاء الاصطناعي**

أدوات الذكاء الاصطناعي الحالية (مثل ChatGPT و Claude و DeepSeek وغيرها) تمتلك قدرات قوية على التعرف على الصور. عندما تواجه عنصر واجهة غير مألوف أو مقطع كود معقد:

1.  **التقط لقطة شاشة**: صوّر الجزء الذي لا تفهمه (مثلاً أيقونة غريبة، أو مقطع تهيئة معقد).
2.  **اسأل**: أرسل الصورة للذكاء الاصطناعي واسأله: "ما هذا؟ ما فائدته؟" أو "ماذا يفعل xxx في هذا الكود؟".
3.  **تعمّق**: إذا كانت إجابة الذكاء الاصطناعي تقنية جداً وصعبة الفهم، استمر في السؤال: "اشرح بلغة بسيطة، يُفضل بمثال من الحياة اليومية."

## 0. مقدمة: لماذا نحتاج IDE؟

في تطوير البرمجيات، يحتاج المبرمجون للقيام بشكل متكرر بكتابة الكود، وإدارة الملفات، والتجميع والتشغيل، وتصحيح الأخطاء. إذا كانت كل هذه العمليات تتطلب برامج منفصلة (مثلاً استخدام المفكرة لكتابة الكود، وسطر الأوامر للتجميع، ومستكشف الملفات لإدارة الملفات)، فستكون الكفاءة منخفضة جداً ومعرضة للأخطاء.

**القيمة الأساسية لـ IDE (بيئة التطوير المتكاملة)** تكمن في **التكامل**. يجمع جميع الأدوات اللازمة لتطوير البرمجيات (المحرر، المُجمّع، المصحح، مدير الملفات، إلخ) في واجهة رسومية موحدة، مما يوفر تجربة عمل شاملة.

**VS Code هو أحد أشهر بيئات التطوير.** رغم أنه في جوهره محرر كود خفيف، إلا أنه بفضل نظام الإضافات القوي، يمتلك جميع الوظائف الأساسية للـ IDE (تحرير الكود، التصحيح، التحكم في الإصدارات، إلخ)، ويُعتبر على نطاق واسع بيئة التطوير المفضلة لتطوير الواجهة الأمامية وال_fullstack الحديث.

باختصار، يهدف الـ IDE لتعظيم إنتاجية المطور وتقليل تكلفة الوقت للتنقل بين الأدوات المختلفة.

> 🔗 **تحميل الموارد**:
>
> - [تحميل VS Code من الموقع الرسمي](https://code.visualstudio.com/Download)
> - [تجربة VS Code على الويب](https://vscode.dev/)
>
> **VS Code (Visual Studio Code)** هو محرر كود مجاني ومفتوح المصدر وعبر منصات طوّرته Microsoft. بفضل خصائصه من **خفة، وإضافات وفيرة، وبدء تشغيل سريع**، أصبح أحد أدوات التطوير الأكثر شعبية في العالم. سواء كنت تكتب Python أو JavaScript أو C++، يمكن لـ VS Code من خلال تثبيت الإضافات أن يصبح "الأداة المثالية" الأنسب لك.

## 1. تحليل الواجهة الرئيسية

واجهة بيئات التطوير الحديثة (باعتبار VS Code مثالاً) مصممة بعناية وعادة تتضمن أربع مناطق رئيسية:

1. **الشريط الجانبي (Sidebar): إدارة الموارد**
   يعرض شجرة ملفات المشروع، ويدعم إنشاء وإعادة تسمية ونقل وحذف الملفات، مما يوفر رؤية شاملة ووصولاً سريعاً لهيكل المشروع.

2. **منطقة المحرر (Editor Area): إنشاء الكود**
   المنطقة المركزية لكتابة وتعديل الكود. تدعم تلوين بناء الجملة، والإكمال التلقائي الذكي، وفحص بناء الجملة، إلخ، مما يوفر بيئة كتابة كود فعالة وذكية.

3. **اللوحة السفلية (Panel): التنفيذ والتغذية الراجعة**
   للتفاعل مع النظام الأساسي وعرض نتائج التشغيل. تتضمن الطرفية (Terminal)، والمخرجات (Output)، إلخ، لتنفيذ الأوامر وعرض السجلات والتصحيح.

4. **شريط الأنشطة (Activity Bar): التنقل بين الوظائف**
   يقع في أقصى يسار الواجهة، يحتوي على أيقونات مستكشف الملفات، البحث، إدارة Git، إلخ، للتبديل السريع بين سياقات العمل المختلفة (مثل "كتابة الكود" و"إدارة التحكم في الإصدارات").

## 2. عرض تفاعلي: تجربة الوظائف

رؤية أفضل من سماع. لتشعر حقاً براحة الـ IDE، أعددنا لك **بيئة VS Code افتراضية**.

**جرّب العمليات التالية**:

1.  انقر على **"▶ بدء الجولة التلقائية"** في الزاوية العلوية اليمنى واتبع المؤشر للتعرف على كل منطقة.
2.  **استكشاف حر**: انقر على الأيقونات يساراً لتبديل العروض، أو انقر على أسماء الملفات لفتح الكود.
3.  **اختبر التكامل**: ستكتشف أن إدارة الملفات، وتحرير الكود، وتشغيل الطرفية كلها متصلة بسلاسة في نافذة واحدة.
4.  **تثبيت الإضافات**: من القائمة المنسدلة، اختر وضع **"تثبيت الإضافات (Extensions)"** لتجربة كيفية تثبيت إضافة Python في متجر افتراضي.

## 3. الآلية الأساسية: لماذا يستطيع VS Code كل شيء؟

قد تتساءل: كيف يمكن لنفس البرنامج أن يُستخدم لكتابة Python و C++ وتطوير الويب؟ كيف يحقق ذلك؟
في الواقع، يمكن تلخيص فلسفة تصميم VS Code في جملة واحدة: **"نواة بسيطة، قدرات قابلة للإضافة".**

### 3.1 النواة البسيطة: مجرد "لوحة رسم"

تخيل أنك حمّلت VS Code للتو؛ إذا لم تثبت أي إضافات، فهو في الواقع **لا يفهم البرمجة**.
في هذه الحالة، هو في جوهره مجرد **محرر نصوص قوي**.

- يتحكم في عرض النص (التصيير).
- يتحكم في إدارة الملفات (الإدخال/الإخراج).
- لكنه لا يعرف أن `print("Hello")` هو كود Python، ولا أن `int main()` هو نقطة الدخول في C++.

### 3.2 نظام الإضافات: حقن "الروح"

لكي يتمكن VS Code من "فهم" الكود، نحتاج لتثبيت **إضافات (Extensions)**.
الإضافات كمترجمين متخصصين:

- **إضافة Python**: تخبر VS Code ما هي المتغيرات، ما هي الدوال، كيفية تشغيل ملفات `.py`.
- **إضافة C++**: تخبر VS Code كيفية استدعاء المُجمّع، وكيفية تصحيح الذاكرة.

هذا التصميم يجعل VS Code خفيفاً جداً — إذا كنت لا تكتب Java، فلست بحاجة لحمل بيئة تشغيل Java.

### 3.3 العملية الداخلية: من الكود إلى التنفيذ

لنرى من خلال سيناريو محدد كيف يتعاون VS Code والإضافات والبيئة الأساسية.
لنفترض أنك كتبت سطر كود Python ونقرت على **تشغيل** أو **تصحيح**:

#### 1. التعرف على اللغة (Activation)

يكتشف VS Code الامتداد `.py` ويُفعّل تلقائياً **إضافة Python**. تستلم الإضافة المحرر، وتبدأ التحليل النحوي، وتلوين الكود (تلوين بناء الجملة)، وتقدم اقتراحات ذكية.

#### 2. تفويض المهام (Delegation)

عند إصدار تعليمات، لا تنفذ الإضافة الكود مباشرة، بل **تفوّض** المهمة لأدوات متخصصة أساسية:

- **وضع التشغيل**: تولّد الإضافة أمراً (مثل `python main.py`) وترسله لـ **الطرفية** في النظام للتنفيذ.
- **وضع التصحيح**: تبدأ الإضافة **محول تصحيح (Debug Adapter)**. هو كـ "مسبار مراقبة" يتصل بمُفسّر Python داخلياً، مما يتيح لك التحكم في تنفيذ الكود سطراً بسطر.

#### 3. تغذية راجعة للنتائج (Feedback)

ينفذ مُفسّر Python (أو المُجمّع) الكود ويُرجع النتائج (أو رسائل الخطأ) للإضافة. تنقلها الإضافة وتعرضها في **لوحة الطرفية السفلية** في VS Code.

### 3.4 الخلاصة: تشبيه "المطعم"

إذا بدت لك الصيغة أعلاه مجردة، يمكننا تشبيه كتابة الكود بـ **الذهاب للأكل في مطعم**:

1.  **VS Code هو "صالة الطعام"**:
    - الديكور فخم والبيئة مريحة (تلوين الكود، سمات جميلة).
    - **لكن الصالة نفسها لا تُنتج الطعام**. أنت تجلس هنا فقط لـ "طلب" (كتابة الكود) بشكل أكثر راحة.

2.  **البيئة (Python/Node) هي "المطبخ"**:
    - هنا يحدث **الطبخ الفعلي (تشغيل الكود)**.
    - إذا لم يكن للمطعم مطبخ (لم تثبت Python)، فستجلس في الصالة حتى الليل دون أن تأكل.

3.  **الإضافات هي "النُّدُل"**:
    - يربطون الصالة بالمطبخ.
    - يفهمون قائمتك ويذهبون للمطبخ ليقولوا: "الطاولة 3 تطلب 'تشغيل main.py'!"
    - عندما يكون جاهزاً، يعيدون النتيجة (الطعام الساخن) إلى طاولتك.

**الخلاصة**:

- VS Code فقط = **صالة بدون مطبخ** (يمكنك الرؤية لكن لا يمكنك الأكل).
- Python فقط = **مطبخ بدون صالة** (يمكنك الأكل، لكن وأنت تجلس على أرض المطبخ؛ تجربة سيئة).
- **VS Code + إضافات + Python = التجربة المثالية.**

<details class="custom-block details" id="vscode-file-menu">
  <summary>File (ملف): فتح/حفظ/إدارة مساحات العمل</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **إنشاء/فتح الملفات**، **فتح مجلدات المشروع (Folder)**، **إدارة مساحات العمل (Workspace)**، **الحفظ والإغلاق**.

> الأكثر استخداماً هو: Open Folder لفتح مشروع؛ Open… لفتح ملف منفرد؛ ثم Save / Save All لحفظ التغييرات؛ وأخيراً Close Editor / Close Folder لإنهاء الجلسة. محتويات مساحات العمل يمكن تعلمها تدريجياً عند كثرة المشاريع.

- **New Text File**: إنشاء مخزن نصي مؤقت غير مُسمّى.
- **New File…**: إنشاء ملف جديد في المشروع (عادة يطلب المسار/التسمية).
- **New Window**: فتح نافذة VS Code جديدة.
- **New Window with Profile**: فتح نافذة جديدة بملف تعريف محدد (مجموعة إضافات/إعدادات)، مناسب لعزل بيئات الدورات/المشاريع المختلفة.
- **Open…**: فتح ملف منفرد للتحرير.
- **Open Folder…**: فتح مجلد كدليل جذر للمشروع (الطريقة الأكثر شيوعاً "لفتح مشروع").
- **Open Workspace from File…**: فتح ملف `.code-workspace` لتحميل مساحة عمل متعددة المجلدات.
- **Open Recent**: وصول سريع للملفات/المجلدات/مساحات العمل المفتوحة مؤخراً.
- **Add Folder to Workspace…**: إضافة مجلد آخر لمساحة العمل الحالية.
- **Save Workspace As…**: حفظ هيكل مساحة العمل الحالية كملف `.code-workspace`.
- **Duplicate Workspace**: نسخ تهيئة مساحة العمل الحالية.
- **Save**: حفظ تغييرات الملف الحالي.
- **Save As…**: حفظ الملف الحالي باسم/مسار جديد.
- **Save All**: حفظ جميع الملفات المفتوحة ذات التغييرات.

- **Share**: مدخل متعلق بالمشاركة/التعاون.
- **Auto Save**: تبديل استراتيجية الحفظ التلقائي.
- **Revert File**: تجاهل التغييرات غير المحفوظة والعودة لنسخة القرص.
- **Close Editor**: إغلاق التبويب الحالي.
- **Close Folder**: إغلاق مجلد المشروع الحالي.
- **Close Window**: إغلاق نافذة VS Code الحالية.

<details class="custom-block details" id="vscode-edit-menu">
  <summary>Edit (تحرير): التحرير الأساسي، البحث والاستبدال، التعليقات</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **التراجع/الإعادة**، **القص/النسخ/اللصق**، **البحث والاستبدال**، **التعليقات وإجراءات التحرير**.

- **Undo / Redo**: "حبوب الندم" عند الخطأ في الكود.
- **Cut / Copy / Paste**: ناقل النصوص.
- **Find / Replace**: البحث أو التعديل المجمّع في الملف الحالي.
- **Find in Files / Replace in Files**: البحث والاستبدال الشامل (في كل المشروع)، قوي جداً لكن استخدمه بحذر.
- **Toggle Line Comment**: `Ctrl + /`، تعليق/إلغاء تعليق السطر الحالي بسرعة.
- **Toggle Block Comment**: `Shift + Alt + A`، تعليق/إلغاء تعليق التحديد بسرعة.
- **Emmet: Expand Abbreviation**: الأداة السحرية لتطوير HTML/CSS، اكتب اختصارات ثم Tab لتوسيع الكود.

</details>

<details class="custom-block details" id="vscode-selection-menu">
  <summary>Selection (تحديد): المؤشرات المتعددة والتحديد الذكي</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **التحكم بالمؤشر**، **التحرير متعدد الأسطر**، **توسيع/تقليص التحديد**. هذه هي الأداة الفائقة في VS Code لتعزيز الكفاءة.

- **Select All**: تحديد كل محتوى الملف الحالي.
- **Expand Selection / Shrink Selection**: الإحساس الذكي بالبنية النحوية، توسيع أو تقليص التحديد تدريجياً.
- **Copy Line Up / Down**: استنساخ السطر الحالي بسرعة.
- **Move Line Up / Down**: `Alt + ↑ / ↓`، تعديل ترتيب أسطر الكود دون قص ولصق.
- **Add Cursor Above / Below**: `Ctrl + Alt + ↑ / ↓`، تفعيل وضع المؤشرات المتعددة، تحرير عدة أسطر في آن واحد.
- **Add Cursor to Line Ends**: إضافة مؤشر في نهاية كل سطر محدد.

</details>

<details class="custom-block details" id="vscode-view-menu">
  <summary>View (عرض): تخطيط الواجهة والتحكم باللوحات</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **إظهار/إخفاء الشريط الجانبي/اللوحات**، **تعديل التخطيط**، **لوحة الأوامر**، **المخرجات ووحدة تحكم التصحيح**.

- **Command Palette…**: `Ctrl + Shift + P` / `F1`، مركز القيادة العام لـ VS Code، ابحث ونفّذ أي أمر.
- **Open View…**: فتح عرض محدد في الشريط الجانبي بسرعة.
- **Appearance**: التحكم في ملء الشاشة، إظهار/إخفاء شريط القوائم، موقع الشريط الجانبي، مستوى التكبير.
- **Editor Layout**: تقسيم المحرر (لأعلى/لأسفل/يسار/يمين) لمقارنة الكود.
- **Explorer / Search / Source Control / Run / Extensions**: التبديل المباشر لعرْض شريط الأنشطة.
- **Problems / Output / Debug Console / Terminal**: التحكم المباشر في محتوى اللوحة السفلية.
- **Word Wrap**: `Alt + Z`، التحكم في التفاف الأسطر الطويلة تلقائياً.

</details>

<details class="custom-block details" id="vscode-go-menu">
  <summary>Go (انتقال إلى): التنقل والقفز في الكود</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **القفز بين الملفات**، **القفز بين الرموز (الدوال/المتغيرات)**.

- **Back / Forward**: التنقل في مواقع المؤشر السابقة كالمتصفح.
- **Switch Editor…**: التبديل السريع بين التبويبات المفتوحة.
- **Go to File…**: `Ctrl + P`، اكتب اسم الملف لفتحه بسرعة.
- **Go to Symbol in Editor…**: `Ctrl + Shift + O`، عرض الدوال/الأصناف/المتغيرات في الملف الحالي.
- **Go to Definition**: `F12`، الانتقال لتعريف المتغير أو الدالة عند المؤشر.
- **Go to References**: `Shift + F12`، عرض أماكن استخدام المتغير أو الدالة.
- **Go to Line/Column…**: `Ctrl + G`، الانتقال لرقم سطر محدد.

</details>

<details class="custom-block details" id="vscode-run-menu">
  <summary>Run (تشغيل): التصحيح والتنفيذ</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **بدء التصحيح**، **إدارة نقاط التوقف**.

- **Start Debugging**: `F5`، تشغيل البرنامج في وضع التصحيح (يدعم نقاط التوقف ومراقبة المتغيرات).
- **Run Without Debugging**: `Ctrl + F5`، تشغيل مباشر دون مُصحح.
- **Stop Debugging**: إنهاء جلسة التصحيح الحالية قسراً.
- **Restart Debugging**: إعادة التشغيل.
- **Toggle Breakpoint**: `F9`، وضع أو إزالة نقطة حمراء (نقطة توقف) على السطر الحالي.
- **New Breakpoint**: يدعم نقاط التوقف المشروطة، ونقاط توقف السجل، إلخ.

</details>

<details class="custom-block details" id="vscode-terminal-menu">
  <summary>Terminal (طرفية): سطر الأوامر المدمج</summary>

هذه القائمة مسؤولة بشكل أساسي عن: **طرفية جديدة**، **إدارة نوافذ الطرفية**.

- **New Terminal**: فتح Shell جديد (PowerShell/Bash/Zsh) في اللوحة السفلية.
- **Split Terminal**: تقسيم الطرفية أفقياً/عمودياً لتشغيل عدة أوامر.
- **Run Task…**: تشغيل مهام البناء/الاختبار المُعرّفة في `tasks.json`.

</details>
